# Move Fuzzer (`move-fuzz`)

[cite_start]Move Fuzzer is a fuzz-testing tool designed specifically for the Move programming language, with a particular focus on the Sui version developed by Mysten Labs[cite: 32, 772]. [cite_start]The primary goal of this project is to identify vulnerabilities, security flaws, and arithmetic errors, such as overflows, within smart contracts[cite: 30, 31, 786]. [cite_start]It achieves this by executing programs with invalid, malformed, or random inputs and monitoring their behavior[cite: 30, 31]. 

[cite_start]Inspired by Rust's `cargo-fuzz` [cite: 34][cite_start], the project integrates the `libfuzzer` engine (part of the LLVM project) with the Move Virtual Machine[cite: 40, 101].

## 🏗️ Project Structure

[cite_start]The project is written in Rust and is divided into two main packages[cite: 58]:
* [cite_start]**CLI Package (`move-fuzz`)**: Manages the command-line interface, handles user requests, initializes projects, compiles Move modules, and configures the execution environment[cite: 58, 59, 60].
* [cite_start]**`move-libfuzzer` Package**: The core component that interfaces with the `libfuzzer` library via FFI (Foreign Function Interface)[cite: 61, 188]. [cite_start]It loads Move modules, receives raw byte data generated by the fuzzer, formats it for the Move VM, and executes the target functions[cite: 170, 171, 172, 173].

## ⚙️ Installation

[cite_start]Installation requires the Rust toolchain[cite: 125]. The process is automated through a Bash script.

1. [cite_start]Ensure `rustc` and `cargo` are installed on your system[cite: 125, 127].
2. Run the integrated setup script:

```bash
   ./setup.sh
```

This script will automatically:

- Detect the host platform.
- Install the Move CLI (`move-cli`) if it is not already installed.
- Compile and install the `move-fuzz` CLI.
- Compile `move-libfuzzer`, automatically setting the necessary flags for the Address Sanitizer (`ASAN_OPTIONS`) and coverage generation (`RUSTFLAGS`).

Note: You can run the script with the `-clean` flag to perform a clean installation by removing existing build files first.

## 🚀 Usage Guide

Interaction with the fuzzer happens via the `move-fuzz` command. Here are the essential commands to start testing your smart contracts:

### 1. Initialization

To set up a new fuzzing project, navigate to your Move project directory and run:

Bash

```
move-fuzz init
```

This command creates a `fuzz` directory, generates a dedicated `Move.toml` file, creates a `.gitignore`, and provides a template for your first test target at `fuzz/sources/fuzz_target_1.move`.

### 2. Adding Targets

To add new fuzz targets to your project:

Bash

```
move-fuzz add --target-name <target_name>
```

### 3. Building

To compile the fuzzing project and its modules:

Bash

```
move-fuzz build --target-name <target_name>
```

### 4. Running the Fuzzer

To start fuzzing a specific target:

Bash

```
move-fuzz run --target-name <target_name>
```

During execution, this command sets up the environment and launches `move-libfuzzer` to continuously generate inputs. You can pass additional `libfuzzer` options directly, such as `-max_len=<len>`, `-runs=<number>`, `-max_total_time=<time>`, or `-timeout=<time>`.

### 5. Debugging and Formatting

If the fuzzer finds a crash, you can test that specific input (or any other file) using the `fmt` command:

Bash

```
move-fuzz fmt --target-name <target_name> <input_file_path>
```

### 6. Advanced Commands

- **`coverage`**: Runs a target and generates a basic code coverage report.
- **`cmin` / `tmin`**: Attempts to minimize the entire test corpus or the size of a single test case, respectively, while maintaining the same code coverage.

## 🎯 Target Identification

There are two primary ways to specify which function the fuzzer should test:

1. **By Generated Target Name**: Ideal for functions created specifically for fuzzing using the CLI. Simply use `--target-name <target_name>`.
2. **Directly on Move Code**: If a standard function accepts parameters that the fuzzer can generate, you can test it directly by specifying both its module and function name : `--target-module <module_name> --target-function <function_name>`
